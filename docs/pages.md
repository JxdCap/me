# 页面与路由架构 (Site Architecture)

本项目采用“单页应用内空间切换”模式，不建议使用传统的页面跳转。

## 1. 空间模型：三层深度系统

本站的页面逻辑建立在 Z 轴的深度关系上：

1.  **顶层 (Overlay Layer)**
    *   **Index (菜单索引)**：全局导航中心，覆盖一切内容。
    *   **Reader (沉浸阅读)**：内容阅读中心，覆盖主站。
2.  **主层 (Main Stage)**
    *   **HomePage**：包含 Header、Hero 和 Memo Stack 的核心展示区。
3.  **底标层 (Grounding Layer)**
    *   **Decorative Footer**：作为页面“锚点”的装饰文字，随空间状态动态隐现。

## 2. 交互状态流转

*   **状态 A：索引态 (Home)**
    *   用户初次进入的平衡状态。
*   **状态 B：聚焦态 (Active Skill)**
    *   点击技能标签。主页面重心微移，页脚悄悄隐去，腾出阅读空间。
*   **状态 C：沉浸态 (Zine Stream)**
    *   点击卡片。主页面背景推后并模糊，进入可以无限滚动的历史河流。

## 3. 未来扩展（策展人门户 / Curator Portal）

虽然目前以公开展示为主，但未来计划加入私密管理区：
*   **身份验证**：通过极简的隐写式入口（如长按 Logo）进入登录。
*   **实时编辑**：直接在页面上以可视化方式编辑大字标题、上传图片。
*   **发布流**：手机端即时拍摄并推送到 Memos 堆叠区。
